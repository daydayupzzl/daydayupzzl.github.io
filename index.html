2.Git 的三种状态与工作模式

  1. 工作状态 git status

    状态 描述
    commiteed 表示数据已保存到本地仓库中
    modified 表示修改了文件，未保存到本地仓库
    staged 表示对已修改文件的当前版本作了标记，便之包含在下次提交的快照中

  2. 工作区

    分类 描述
    工作区 比如创建的本地目录
    暂存区 Git的版本库中存了很多东西，其中有暂存区，还有 Git 自动创建的第一个分支 master,以及指向master的指针HEAD
    Git仓库 工作区中有一个隐藏目录 .git ，这个不属于工作区，而是Git的版本库

  3. git的工作流程

    • 在工作区中修改某些文件 [modified]
    • 对提交后的文件进行快照，后添加到暂存去 [add]
    • 提交更新，将保存在暂存区的文件快照永久存储到git 仓库中 [commit]
    • 推送到远程仓库 [push]

3.2 比较差异

执行 git diff HEAD --test01.txt 与版本库内容进行比较,差异比较说明如下：

— 表示变动前的文件，+++ 表示变动后的文件

变动的位置用两个@作为起首和结束

@@ -1,2 +1,3 @@ ：

​ 减号表示第一个文件，1表示第一行，2表示连续的两行;

​ 加号表示变动后，称为第二个文件从第一行开始的的连续三行

3.3 后悔add暂存

​ git reset HEAD 或 git restore --staged <file>

3.4 操作

**简化 git-log 输出 **

git log -x --pretty=oneline

版本回退

回退x个版本：git reset --hard HEAD~x

转到未来版本

查看唯一标识

Git只关注文件是否被修改过（添加，删除，更新）

如果工作区误删文件，可以用git checkout 来将本地git仓库的文件拷贝到工作区[前提本地仓库存有本文件]。

或 git restore

如果确定删除，执行git rm命令即可

5.Git 分支操作

多人协作开发时，通常不会直接在主干master上进行操作，而是重新开辟新的分支进行开发调试工作，当项目调试通过后，才会将分支项目的代码合并到主干中。

5.1基本命令

命令 描述
git checkout branch 切换到指定的分支
git checkout -b new_branch 新建分支并切换到新建分支
git branch -d branch 删除指定分支
git branch 查看本地所有分支
git branch -a 查看本地与远程所有分支
git merge branch 合并分支 [只能在分支上合并主干，不能在主干上合并分支]
git branch -m | -M oldbranch newbranch 重命名分支，如果newbranch名字已存在，则需要使用-M强制重命名，否则，使用-m进行重命名

举例：

  • 新建分支

    创建分支时，会将主干上的所有文件对应到分支上

  • 切换分支，查看所有分支，合并分支

    只能在分支上合并主干，不能在主干上合并分支

  • 删除分支

  • 分支重命名

5.2分支push和pull操作

命令 描述
git branch -a 查看本地和远程分支
git push origin branch_name 推送本地分支到远程仓库
git push origin :remote_branch 删除远程分支（本地分支还保留）
git checkout -b local_branch origin/remote_branch 拉取远程指定分支并在本地创建分支

举例：

  • 查看本地&远程分支

  • 推送分支

  • 删除远程分支

  • 拉取远程指定分支，并在本地创建分支

    场景：一般到公司都是半路跳槽，接手的都是开发到一般的项目，但是都是在分支上开发的，所以必须拉取远程仓库中的分支，并在本地创建该分支进行开发。

5.3分支操作冲突出现与解决

开发时对不同分支下同一个文件修改后，执行合并时会出现文件修改冲突

  1. 本地分支操作冲突

    • 制造冲突

    • 冲突详情

    • 冲突解决

    • 查看分支合并图 git log --graph --pretty=oneline

  2. 多人协作操作冲突

    拉取远程库dev并在本地创建dev开发库，执行命令git checkout -b dev origin/dev 这里以同台机器不同的窗口模拟两个用户

    举例：

    • 模拟冲突

6.标签管理

Git 可以对某时间点上的发行版本打上标签。

开发中在发布某个版本时，通常使用版本库软件命令来对某一个版本打上一个标签，以便标识

基本命令：

命令 描述
git tag tag_name 新建标签【默认为HEAD，即最后一次提交的描述信息】
git tag -a tag_name -m ‘xxx’ 新建标签，并指定标签的描述信息
git tag 查看所有标签
git tag -d tag_name 删除一个本地标签
git push origin tag_name 推送指定本地标签到远程仓库
git push origin –tags 推送全部未推送的本地标签到远程
git push origin :refs/tags/tag_name 删除一个远程标签

举例：

  • 查看标签，新建标签

  • 推到远程仓库

  • 删除一个远程标签

7.SSH

  1. Git支持的协议

    • https 协议
    • SSH (Secure Shell)
    • local (本地)
    • GIt 协议

    码云和GitHub仅支持Https和SSH两种协议

  2. SSH协议

    Secure Shell 安全外壳协议,利用本协议可以防止远程管理过程中的信息泄露.

    由于本地与远程仓库之间的传输都是通过 SSH 加密的,所以必须让远程仓库认证你的 SSH key ,在此之前,必须要生成SSH key

    使用SSH协议通信时,建议使用密钥的验证方式,必须为自己创建公私钥,并把公钥放在远程仓库平台上.私钥放在自己本地客户端上.

    之前的 Https,是采用 username+password 形式验证身份

  3. 操作步骤

    • 使用Git命令行工具,命令:ssh-keygen-t rsa 生成密钥 [一路下一步]
    • 公私钥位置: c:\Users\张祚龙\ .ssh 目录下
    • 复制公钥文件内容到码云平台上
